# ASCII Art Editor Extension - Architecture & Requirements

## 1. プロジェクトの目的
VS Code上でアスキーアート（AA）を快適に作成・編集できる拡張機能を開発する。

## 2. コアアーキテクチャ（最重要）
* **アプローチ:** `CustomTextEditor` API（Webview）は**絶対に使用しない**。
* **採用技術:** VS Code標準のテキストエディタ（Monaco Editor）と、専用の言語拡張子（`.ascii`）の組み合わせで実装する。
* **理由:** Webview上で独自にエディタを構築した場合、日本語入力（IME）の変換状態（composition）の同期やUI制御が極めて困難になり、変換の強制キャンセル等の致命的なバグを引き起こすため。入力処理はすべてVS Code本体に委ねる。

## 3. 基本仕様（言語設定）
* 拡張子: `.ascii`
* フォント: 強制的に等幅フォント（Monospace）を適用する。

## 4. 文字幅の計算制約（East Asian Width）
AAのレイアウトを維持するため、テキスト操作時は常に文字の「表示幅」を計算すること。
* 半角英数字・半角記号・半角スペース: **幅 1**
* 全角文字（日本語、全角記号など）: **幅 2**
* ※ `string-width` 等のライブラリを使用、または正規表現等で文字幅を正確に判定するロジックを共通関数として実装すること。

## 5. 実装する機能

### 機能A: キャンバスの生成
* **コマンド:** `ascii-art.createCanvas`
* **動作:** コマンドパレットから実行した際、80文字（半角スペース）×20行で埋められた「未保存の新規ドキュメント（Untitled）」を生成して開く。
* **言語ID:** 開いたドキュメントは自動的に `ascii-art` モードになること。

### 機能B: 矩形選択からの枠線生成
* **動作:** ユーザーが複数行・複数列を矩形選択（`Shift+Alt+Drag` 等）した状態で専用コマンドを実行すると、選択範囲の境界線を罫線文字（`┌`, `─`, `┐`, `│`, `└`, `┘`）に置換して枠線を描画する。

### 機能C: 枠線の遅延フォーマット（自動修整）
**🚨 警告: リアルタイムの `type` コマンドフックによる自動修整は、IME入力と競合して変換を破壊するため禁止する。**
* **動作:** ユーザーが枠内でテキストを編集（特に全角文字を入力）して枠線（右側の `│` など）がズレた場合、**Enterキー押下時**（または専用のアラインメントコマンド実行時）に、枠内の各行の文字幅を再計算する。
* **修整ロジック:** 最も幅の広い行（または元の枠線の幅）に合わせて、足りない幅の分だけ半角スペースをパディングし、右側の枠線（`│`）の位置を縦一列に揃え直す（Markdownのテーブルフォーマッターと同様の遅延評価方式）。
